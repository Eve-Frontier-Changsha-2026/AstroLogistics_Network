# 專案名稱：AstroLogistics_Network (跨星系儲存與全域燃料物流網)

> 📜 **[EVE Frontier 專案憲法與開發準則](https://github.com/Eve-Frontier-Changsha-2026/Constitution/blob/master/EVE_Frontier_Project_Constitution.md)**
> 本專案的世界觀設定與底層相依資源，均遵從此憲法文檔之規範。


## 📌 概念簡介
這個專案旨在打造 EVE Frontier 中的「宇宙級 UPS 與燃料庫」。我們利用 **Smart Assemblies** 將儲存箱 (Storage) 轉化為 Sui 上的 Shared Object，並融合燃料與能量經濟，建立一個橫跨星系的去中心化物流、能源配送、物資轉運和點對點的任務網。

## 🎯 解決痛點與核心循環
- **物資與燃料跨星系傳送**：玩家將貨品放入 A 星系 Smart Storage，透過上鏈與扣除高階燃料，在 B 星系解鎖提領。同時支援燃料的儲存、轉換、販售或借貸。
- **快遞任務市場**：低階物流可產生「快遞任務合約 (Courier Contract)」，交由其他運輸玩家實飛完成，儲存箱建造者從中抽取手續費分潤。
- **工業採集與製造**：玩家採集原礦後，可在燃料網絡的工業節點進行精煉成燃料品級資源，供應龐大艦隊所需。
- **戰略衝突點（摧毀）**：戰略上可以攻擊對手的關鍵燃料樞紐，造成其整個戰線能源短缺；系統同時記錄燃料戰爭的鏈上歷史。

## 🔗 與 Sui / World Contracts 的結合
- 每個燃料桶、電池、倉儲箱用 Sui 物件表示，擁有明確所有權與狀態（容量、品質、位置）。
- 使用 World Contracts 為樞紐寫邏輯：例如依照庫存自動調整價格（AMM 式）、支援「能源借貸」，以未來產量抵押，條件紀錄上鏈。
- 外部 dApp 使用 @evefrontier/dapp-kit 的 GraphQL / hooks 繪製「燃料熱度圖」及「最佳補給路線」。
- 支援 zkLogin + sponsored gas，玩家用 email 登入即可查詢與下單。

## 🗺️ 現實世界 → 遊戲世界 情境映射

本專案的每個模組都對應現實物流產業的實際角色與流程，再投射到 EVE Frontier 的宇宙場景中：

### 倉儲系統 (Storage Module)

| 鏈上設計 | 現實世界 | EVE 宇宙 |
|----------|----------|-----------|
| `Storage` (Shared Object) | 物流倉庫 / 轉運中心 | 星系中的 Smart Storage 建築 |
| `AdminCap` | 倉庫經營許可證 — 可轉讓即「賣掉這間倉庫」 | 站主控制權 |
| `DepositReceipt` | 提貨單 / 倉單 — 誰持單誰提貨，遺失即無法提領 | 高風險宇宙的殘酷代價 |
| `Cargo` wrap 進 Storage | 貨物入庫歸倉庫保管，不在外部流通 | 物資存入 Smart Storage |
| `fee_rate_bps` | 倉儲費率，業主自訂 | 站主經營策略 |

> 現實邏輯：寄件人拿到提貨單 → 可自行提貨，也可交給快遞員代提 → 自然銜接快遞任務市場。

### 威脅情報系統 (Threat Oracle Module)

| 鏈上設計 | 現實世界 | EVE 宇宙 |
|----------|----------|-----------|
| `ThreatMap` | 航運風險評估機構（如 Lloyd's 戰爭險區域列表） | 星系威脅熱度圖 |
| `danger_index` | 海盜高風險海域評級 | 近期戰鬥 / 擊殺頻率 |
| 時間衰減 | 風險評級定期更新，過期事件權重降低 | 舊戰鬥影響逐漸消退 |
| `ReporterCap` | 授權情報員 / 認證安全顧問 | 快遞員回報途中遭遇 |
| `OracleCap` | 中央情報彙整機構 | 系統管理者批量校正 |

> 現實邏輯：跑過危險航道的快遞員掌握一手情報 → 情報本身是有價商品 → 對接憲法「資訊不對稱」原則。

### 跨星系運費公式

| 設計元素 | 現實映射 |
|----------|----------|
| `ln(1 + distance)` 對數衰減 | 長途幹線物流單價更低（規模效應） |
| `cargo_weight` 貨重因子 | 輕量文件 vs 重型貨櫃，運費天差地別 |
| `danger_multiplier` 危險溢價 | 戰區 / 海盜海域的運輸保險加價 |
| 雙路線選擇（安全 vs 危險） | 繞行好望角（安全但遠） vs 亞丁灣直航（近但海盜多） |

> 公式：`fuel_cost = base_rate × ln(1 + distance) × cargo_weight × (1 + β × max_danger) × tier_multiplier`
> 鏈下計算 → 玩家確認 → 鏈上存證，所有傳送都有等價成本。

### 四級物流光譜（價格 vs 時間 Trade-off）

| 等級 | 費用倍率 | 等待時間 | 現實映射 | EVE 映射 |
|------|----------|----------|----------|----------|
| Instant（瞬間） | ×3.0 | 0 min | FedEx 同日達 / 空運 | 量子傳送（土豪用錢買時間） |
| Express（快速） | ×1.5 | 5 min | 隔日達 / 快遞 | 高能躍遷 |
| Standard（標準） | ×1.0 | 15 min | 標準配送 / 陸運 | 常規躍遷 |
| Courier（實飛） | 最便宜 | 不定 | 海運 / 經濟配送 | 委託其他玩家駕船運送 |

> 現實邏輯：用錢買時間是物流業最基本的 trade-off。玩家的選擇本身就在表達他們對「時間 vs 金錢」的偏好 — 這是現實經濟學最核心的概念之一。

### 燃料站經濟 (Fuel Station Module)

| 鏈上設計 | 現實世界 | EVE 宇宙 |
|----------|----------|-----------|
| `FuelStation` | 加油站 / 油庫 | 星系燃料站建築 |
| 庫存反比定價 | 油價隨庫存波動（OPEC 減產 → 油價漲） | 燃料稀缺時自動漲價 |
| `SupplierReceipt` | 供應商送貨單 + 分潤合約 | 補貨者的收益憑證 |
| 緊缺獎勵 ×1.5 | 戰時物資運送的危險津貼 | 低庫存補貨分潤暴漲 |
| 站主抽成 `owner_fee_bps` | 加盟站品牌授權費 / 管理費 | 站主基礎收益 |

> 現實邏輯：燃料站快空了 → 售價飆升 → 補貨分潤暴漲 → 吸引供應商跑來補貨 → 價格回落。自平衡的供需循環。

### 快遞任務市場 (Courier Market Module)

| 鏈上設計 | 現實世界 | EVE 宇宙 |
|----------|----------|-----------|
| `CourierContract` | 物流委託單（貨拉拉 / Uber Freight） | 快遞任務合約 |
| 發單人押金 | 寄件人預付運費 + 取消違約金 | 防止濫發任務 |
| 接單人押金 | 快遞員保證金 / 貨運保險 | 防止接了不送 |
| `CourierBadge` | 快遞員工牌 — 一次性授權 | 憑此從倉庫代提貨物 |
| 五階段生命週期 | 下單→接單→取貨→運送→送達 | 完整物流追蹤 |
| 超時賠付 | 物流超時賠付條款 | 押金自動賠給對方 |
| 完成任務獲 `ReporterCap` | 跑過一趟獲得情報員資格 | 有資格回報航路威脅 |

> 現實邏輯：雙邊押金是「信號機制」而非「罰款機制」— 願意押錢代表認真，讓陌生人之間不需要信譽系統就能交易。

### 跨星系傳送 (Transport Module)

| 鏈上設計 | 現實世界 | EVE 宇宙 |
|----------|----------|-----------|
| `TransportOrder` | 國際貨運提單 (Bill of Lading) | 跨星系傳送訂單 |
| 鏈下算價 → 玩家確認 → 鏈上存證 | 物流公司報價 → 客戶簽約 | 燃料消耗確認 |
| `route` 路徑記錄 | 貨物追蹤號 + 途經城市 | 途經星系清單 |
| `danger_snapshot` | 下單時的航線風險快照 | 價格鎖定依據 |

> 現實邏輯：報價基於當下風險評估，一旦客戶接受就鎖定價格，即使航線之後變更危險也不影響已簽約的訂單。

## 🏆 得獎潛力
- **完美對齊官方願景**：高度貼合「A Toolkit for Civilization」，能源與物流是文明基礎世界經濟中樞。
- **具備全域性**：物流路線必然跨星系，所有玩家都會受影響。
- **長期演化潛力**：之後可擴充碳排放、能源稅、封鎖制裁等玩法，形成富有敘事性的經濟戰。
